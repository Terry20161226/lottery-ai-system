## 🔍 问题诊断

### 核心问题：自适应逻辑过于激进

**1. 连号率过高（76.3%）**
- 虽然超过 70% 目标，但可能过度拟合
- 连号过多导致号码组合单一
- 降低了多样性，错失大奖

**2. 动态 ε 效果不佳**
- 连胜时增加探索（ε=0.15）
- 但连胜时应该保持利用（热号）
- 逻辑可能反了

**3. 动态投注未发挥作用**
- 平均注数 1.95 注 vs 固定 2 注
- 差异太小，未体现自适应优势

## 🎯 优化建议 v4.1

### 方案 A：调整自适应参数

```python
# 更保守的自适应
def get_adaptive_epsilon(self):
    win_rate = self.get_recent_win_rate()
    
    if win_rate > 0.15:
        return 0.05  # 连胜：减少探索，保持热号
    elif win_rate < 0.08:
        return 0.02  # 连败：专注热号
    else:
        return 0.08  # 正常

def get_adaptive_consecutive_prob(self):
    # 降低连号概率上限
    if self.win_streak >= 2:
        return 0.75  # 连胜：75% 连号
    elif self.lose_streak >= 3:
        return 0.60  # 连败：60% 连号
    else:
        return 0.70  # 正常：70%
```

- 连号率：76.3% → 65-70%
- 收益率：-81.45% → -72% 左右
