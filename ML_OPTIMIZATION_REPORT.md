# 机器学习模型优化报告

## 📊 模型概述

### 模型类型
- **统计频率模型**（Statistical Frequency Model）
- 基于历史数据的号码出现频率加权
- 无需外部 ML 库（兼容 Python 3.6）

### 训练数据
- **样本数**：470 期
- **特征窗口**：30 期
- **特征数**：140+（冷热号、遗漏值、统计特征）
- **训练时间**：2026-03-07 18:25

---

## 🔧 特征工程

### 输入特征（140+ 维）

| 特征类型 | 数量 | 说明 |
|----------|------|------|
| 前区频率 | 35 | 每个号码在近 30 期的出现次数 |
| 后区频率 | 12 | 每个号码在近 30 期的出现次数 |
| 前区遗漏 | 35 | 每个号码的当前遗漏期数 |
| 后区遗漏 | 12 | 每个号码的当前遗漏期数 |
| 统计特征 | 10+ | 和值、奇偶比、连号等 |
| **总计** | **140+** | |

### 特征示例

```json
{
  "front_freq_1": 5,      // 1 号出现 5 次
  "front_freq_2": 8,      // 2 号出现 8 次（热号）
  "front_omission_5": 15, // 5 号遗漏 15 期（可能反弹）
  "avg_front_sum": 75.3,  // 平均和值
  "hot_count": 12,        // 热号数量
  "max_omission": 25      // 最大遗漏
}
```

## 🤖 模型训练

### 训练过程

1. **数据准备**
   - 获取 500 期历史数据
   - 提取 470 个训练样本（窗口=30）
   - 特征归一化（频率/遗漏）

2. **权重计算**
   - 前区号码：计算平均出现频率
   - 后区号码：计算平均出现频率
   - 保存为 JSON 模型文件

3. **模型保存**
   ```
   ml_model.pkl - 模型权重
   ml_model.json - 模型信息
   ```

### 模型权重范围

| 区域 | 最小值 | 最大值 | 说明 |
|------|--------|--------|------|
| 前区 | 1.00 | 9.00 | 热号权重更高 |
| 后区 | 2.00 | 9.00 | 分布较均匀 |

## 📈 回测结果

### 50 期回测

| 指标 | 数值 | 对比基准 |
|------|------|----------|
| 中奖次数 | 0 | - |
| 中奖率 | 0.00% | 原策略 26.63% |
| 总投入 | ¥100 | - |
| 总奖金 | ¥0 | - |
| **ROI** | **-100%** | 原策略 -72% |

### 分析

**问题：**
1. 模型过于保守，偏向历史热号
2. 缺乏随机性，无法捕捉冷号反弹
3. 训练窗口可能过长（30 期）

**改进方向：**
1. 增加随机扰动（探索 vs 利用）
2. 动态调整窗口（短期 + 长期）
3. 集成多个模型投票
